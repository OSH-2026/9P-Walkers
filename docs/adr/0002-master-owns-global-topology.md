# 主机侧维护权威拓扑

每个 ESP32 coordinator 维护自己直连 STM32 子树的权威身份、地址、链路和路由视图，
并从中派生下发给节点的转发表。多主机模式下，leader 聚合各 owner host 的节点摘要并
分配全局 `mcuN` 名称，但不接管其他 owner 的 UART 数据面。

STM32 只维护本地 peer 状态、upstream 和主机下发的简化转发表，不同步完整拓扑，也
不运行全局路由算法。这个边界把路径重算和诊断留在资源更充足的主机侧，同时保留
owner host 对本地物理链路的唯一控制权。

**状态**：accepted，2026-07-01 按多主机实现更新

**考虑过的方案**：所有节点维护完整拓扑图、所有节点只维护下一跳表、主机维护全局拓扑图并下发从机转发表。最终选择第三种，因为它兼顾主机的全局可见性和从机侧实现的静态内存约束。
