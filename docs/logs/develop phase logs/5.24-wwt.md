## 添加最后一层胶水——节点注册

最开始我的提交为：

>  Add node_connector.c which initialises the ESP32 UART transport at
>    115200 baud (matching the STM32F411 slave), creates an m9p_client on
>    top of it, registers the client as route "mcu1" in cluster_vfs, and
>    attempts TATTACH at boot.  A failed attach (slave not yet connected)
>    is non-fatal; the user can retry via vfs.attach("mcu1") in the Lua
>    shell.

dyh 提交的mesh runtime cluster 是经由 `pwos-shared` 设计的，自动创建 `mesh` 节点网络视图

但是逻辑存在部分问题，修复bug：

1. sync_registered_node 中第二次 find_slot_by_uid 无 null 检查，存在潜在崩溃
2. claim_dispatch 是非原子的 check-then-set，在 ESP32-P4（双核 RISC-V）上存在竞态
3. client_request 对 EBUSY 不重试，导致 poll 任务占用接收链路时所有 VFS 读写立即失败

并且移除我自己写的裸mini9P node connector (否则有冲突，需要走正确的路由)
