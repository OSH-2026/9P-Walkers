# pc_master_emulator 调试记录 5-25

## 背景

本次联调对象是 `pwos-slave` 通过 USART2 与 PC 侧 `tools/pc_master_emulator` 通信。

从机侧：

- 固件通过 ST-Link 烧录。
- 运行时使用 `PWOS_ENABLE_MINI9P_SERIAL` 和 `PWOS_BOARD_ZGT6`。
- Mesh/Mini9P 通信口是 `USART2`，`PA2=TX`，`PA3=RX`，波特率 `1000000`。

PC 侧：

- `pc_master_emulator` 打开 `/dev/ttyUSB0`，波特率 `1000000`。
- 主机等待从机 `REGISTER`，发送 `ASSIGN mcu1 addr=0x11`，再通过 `/mcu1/...` 发起 `cluster_vfs_attach()`。

## Bug 1：打开主机后 reset，从机小灯不再闪

现象：

```text
不开 pc_master_emulator 时，reset 后 PF9 小灯正常慢闪。
启动 pc_master_emulator 后，再 reset，从机小灯不再闪，主机也收不到 REGISTER。
```

最初怀疑是 REGISTER 只发一次导致主机错过。后来确认真正问题是 PC 打开串口时，USB-TTL 的 modem 控制线状态影响了板子 reset/boot。

根因：

- `pc_master_emulator` 打开 `/dev/ttyUSB0` 后没有处理 DTR/RTS/HUPCL。
- 某些 USB-TTL 模块打开串口时会改变 DTR/RTS。
- 如果这些控制线接到板子的 NRST/BOOT0，或经过自动下载电路影响启动，reset 后 MCU 可能无法正常进入应用。

修复：

- 在 `tools/pc_master_emulator/main.c` 的串口初始化里关闭 `HUPCL`。
- 打开串口后通过 `ioctl(fd, TIOCMBIC, ...)` 清掉 `DTR` 和 `RTS`。

效果：

- PC emulator 保持打开时，reset 后从机仍能正常运行。
- 主机可以收到：

```text
[mesh→PC] REGISTER src=0xff dst=0xff seq=1 payload=15
```

硬件注意事项：
- 该bug可能是开发板的特殊问题，因为stm32f411最小板没有出现此bug
- UART 联调只需要接 `TX/RX/GND`。
- 不要接 USB-TTL 的 `DTR`、`RTS` 到板子的 `NRST` 或 `BOOT0`。

## Bug 2：收到 REGISTER 并发送 ASSIGN 后，attach 失败

现象：

```text
[mesh→PC] REGISTER src=0xff dst=0xff seq=1 payload=15
[PC→mesh] ASSIGN mcu1 addr=0x11 (34 bytes)
mesh runtime: bootstrap REGISTER assigned src=0x11
[PC→mesh] next_hop=0x11 30 bytes
cluster_vfs_attach(mcu1) failed: -6
```

含义：

- 主机已经收到 bootstrap REGISTER。
- 主机已经发送 ASSIGN。
- 主机也已经发出 attach 对应的 mesh MINI9P 帧。
- 失败点是没有收到从机的 Rattach 响应。

根因：

- PC 发送 ASSIGN 后立即把 bootstrap REGISTER 映射成 `src=0x11` 并继续 attach。
- 从机处理 ASSIGN、更新本地 mesh 地址到 `0x11` 需要一个很短的时间窗口。
- 如果 PC 立刻发送发往 `0x11` 的 attach，从机可能还处于 `local_addr=0xff` 状态，导致不处理这帧，也就不会返回 Rattach。

修复：

- 正式修复策略是：从机在处理 ASSIGN 并确定本地地址后，主动重传一次 REGISTER。
- 主机不能仅凭自己发送了 ASSIGN 就确认节点已连上；必须等收到从机以已分配地址发出的 REGISTER，才把该节点确认进入可 attach 状态。
- 主机侧 client 增加固定次数重试：attach/read 等同步请求如果遇到短暂超时或 busy 类错误，可以按固定次数重新请求，超过次数后才向上层返回失败。

说明：

- `usleep(100000)` 只能缓解竞态，不是最终协议语义。
- 最终语义必须由“从机已切换地址并重发 REGISTER”来确认。

效果：

- 避免 ASSIGN 和 attach 之间的竞态。
- 主机不会在从机仍处于 `local_addr=0xff` 时提前发起 attach。
- client 固定次数重试可以吸收 UART 半包、短暂超时、节点刚切地址等瞬时问题。

## 当前正确流程

```text
PC 打开串口，关闭 HUPCL/DTR/RTS
从机 reset
从机发送 REGISTER src=0xff
PC 收到 REGISTER
PC 发送 ASSIGN mcu1 addr=0x11
从机处理 ASSIGN，切换本地地址到 0x11
从机以 src=0x11 重发 REGISTER
PC 收到 src=0x11 REGISTER 后确认 mcu1 在线
PC 发起 cluster_vfs_attach("mcu1")
如果 attach 短暂失败，client 按固定次数重试
从机响应 Rattach
后续通过 /mcu1/... 访问
```

bash pwos-slave/build.sh flash-test
