# 9P-Walkers 答辩五人分工研究任务

> 本文件将项目仓库拆分为五份相对独立、覆盖答辩高频追问细节的任务包。每份任务包含：负责模块、必读源码路径、需重点掌握的知识点、可能被追问的问题清单。

| 人员 | 负责领域 | 核心定位 |
|---|---|---|
| 成员 A | 底层协议与链路层 | link frame v2、CRC、parser、mesh2 控制面协议 |                                   霍斌

## 成员 A：底层协议与链路层

### 负责内容
- 链路帧 v2 的完整格式与编解码
- CRC-16/CCITT-FALSE 实现
- 流式字节流解析器状态机
- mesh2 控制面各消息结构
- 共享模块的测试用例

### 必读源码文件

#### link frame 与 CRC
- `pwos-shared/link/pwos_link_frame.h`
- `pwos-shared/link/pwos_link_frame.c`
- `pwos-shared/link/pwos_link_crc.h`
- `pwos-shared/link/pwos_link_crc.c`
- `pwos-shared/link/pwos_link_parser.h`
- `pwos-shared/link/pwos_link_parser.c`

#### mesh2 控制面
- `pwos-shared/mesh2/pwos_mesh2_control.h`
- `pwos-shared/mesh2/pwos_mesh2_control.c`

#### 测试
- `pwos-shared/link/tests/test_pwos_link.c`
- `pwos-shared/mesh2/tests/test_pwos_mesh2_control.c`

### 必须掌握的细节
1. link frame 总长度、头部长度、payload 最大长度分别是多少？
2. 帧格式中 magic、version、hdr_len、type、src、dst、ttl、seq、ack、payload_len、hdr_crc、payload_crc 各占多少字节？
3. 为什么用双层 CRC？头 CRC 覆盖哪些字节？payload CRC 覆盖哪些字节？
4. parser 状态机有几个状态？找 magic、头部校验、payload 接收、最终解码分别在什么条件下触发？
5. parser 出错后如何重新同步？保留 magic 前缀策略是什么？
6. 特殊地址 0x00 和 0xFF 分别代表什么？
7. mesh2 控制面有哪些消息？每种消息长度是多少？
8. `NODE_REGISTER` 和 `ADDR_ASSIGN` 的字段含义？
9. `LINK_STATE` 包含哪些字段？metric 怎么算？
10. `ROUTE_UPDATE` 中 action 的 SET/DELETE 含义？route_version 作用？

### 可能被追问的问题
- 为什么 payload 最大是 512 字节？
- 如果噪声导致 magic 出现在 payload 里会不会误切？
- TTL 字段在哪里递减？在哪几个文件里处理？
- 为什么用 CRC-16/CCITT-FALSE 而不是其它 CRC？
- mesh2 消息为什么是固定长度？这对 MCU 有什么好处？

## 建议协作方式

1. 每人先独立阅读自己负责的源码和文档。
2. 每晚集中讨论 30 分钟，互相讲解自己模块的关键逻辑。
3. 交叉提问：成员 A 问成员 D parser 如何被调用，成员 D 问成员 A CRC 细节。
4. 每人准备 3 张 PPT 摘要，写明自己模块的输入、输出、关键函数、易错点。
5. 答辩前模拟一次完整 Q&A，互相挑战细节。
